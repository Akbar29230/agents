# Как задеплоить с бэкендом на Vercel

## Структура

Добавь в свой репозиторий на GitHub эти два файла:

```
твой-репозиторий/
├── api/
│   └── chat.js       ← бэкенд-прокси (положить именно в /api в корне репо)
└── src/
    └── App.jsx        ← замени свой текущий компонент этим файлом
```

`api/chat.js` — обязательно в корне репозитория, в папке `api`. Vercel сам превращает файлы из этой папки в serverless-функции — ничего дополнительно настраивать не нужно.

## Шаги

1. **Скопируй файлы в репозиторий**
   - `api/chat.js` — прокси, который держит API-ключ на сервере
   - `App.jsx` — обновлённый компонент (обращается к `/api/chat` вместо `api.anthropic.com` напрямую)

2. **Закоммить и запушь в GitHub**
   ```
   git add .
   git commit -m "Добавлен бэкенд-прокси для Claude API"
   git push
   ```

3. **Подключи репозиторий к Vercel**
   - Зайди на vercel.com → New Project → выбери свой GitHub-репозиторий
   - Framework Preset определится автоматически (Vite/CRA/Next — смотря что у тебя)

4. **Добавь переменную окружения**
   - В настройках проекта на Vercel: **Settings → Environment Variables**
   - Добавь: `ANTHROPIC_API_KEY` = твой настоящий ключ с console.anthropic.com
   - ⚠️ Никогда не клади ключ прямо в код или в GitHub — только через переменные окружения

5. **Deploy**
   - Vercel задеплоит автоматически при пуше. Или вручную: `vercel --prod`

## Проверка

После деплоя открой `https://твой-проект.vercel.app` — агенты должны работать, запросы теперь идут через твой сервер, ключ нигде не светится в браузере.

Если получишь ошибку — открой консоль браузера (F12), там будет текст ошибки от `/api/chat`, обычно это подскажет, не забыл ли добавить `ANTHROPIC_API_KEY` в Vercel.
